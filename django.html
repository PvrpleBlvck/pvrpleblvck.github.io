What is Django?

Django is the name of a gypsy jazz guitarist.

It is also a backend framework, written in python, for building database-driven Web applications.

  • Models: Django comes with an object-relational mapper which you can use to describe your database layout. It works with databases like PostgreSQL, MySQL and Oracle.

  • URLs: Django gives you a clean and elegant way to define your URL routing without the cruft of .php or .asp.

  • Views: Views handle the Http requests routed via the URL scheme.

  • Templates: Django has a template system that helps you display specific information based on the occasion and situation.

Making a Django Project

A Django project is a Python package – i.e. a directory of code – that contains all the settings for an instance of Django. This includes database configuration, Django-specific options and application-specific settings.

Run the following command to auto-generate code to set up a basic Django Project:

$ django-admin startproject traction_site

Django Project Directory Structure

The startproject command creates a traction_site directory. Here's what startproject created:

      traction_site/
        manage.py
        traction_site/
          __init__.py
          settings.py
          urls.py
          wsgi.py
  • The outer traction_site/ root directory is just a container for your project.
  • manage.py: A command-line utility for DB migrations, running the development server among other things.
  • The inner traction_site/ directory is the actual Python package for your project.
  • traction_site/__init__.py: An empty file that tells Python that this directory is a Python package.
  • traction_site/settings.py: Settings/configuration for the Django project.

  • traction_site/urls.py: The URL declarations for this Django project and their associated views.
  • mysite/wsgi.py: An entry-point for WSGI-compatible web servers to serve your project.

Running the Development Server

Let’s verify your Django project works by running the development server:

$ python manage.py runserver

You can see the server is running by opening http://127.0.0.1:8000/ in your browser.

By default, the server's port is 8000 but if you want to change it, specify the port number after runserver in the command line:

$ python manage.py runserver 8080

This command starts the server on port 8080.

What about applications?

Didn't you say in the beginning that Django is for developing web applications????

Now that you have an set up your environment – aka “project” – is set up, you can start creating applications.

What’s the difference between a project and an app? A Web application does something – e.g., a blogging platform, a database of patents or a simple poll app. A project is a collection of configuration and apps for a particular website.

A project can contain multiple apps. An app can be in multiple projects. This modularity is a very powerful feature of Django. It allows you to easily add and remove apps/functionality for your website. Once you write one app, you can use it for other projects.

Creating an app

Each application you write in Django consists of a Python package that follows a certain convention. Django comes with a utility that automatically generates the basic directory structure of an app, so you can focus on writing code rather than creating directories.

To create an app, go to the same directory as manage.py and type this command:

$ python manage.py startapp case_studies

App routing in a Project

To add the case_studies route to your website, point to the root URLconf at the case_studies.urls module. Open traction_site/urls.py and modify it so you have:

urlpatterns = [
    url(r'^case_studies/', include('case_studies.urls')),
    url(r'^admin/', admin.site.urls),
]

Now run the server and open the url http://localhost:8000/case_studies/ in the browser. You should see the text “Hello, world. You’re at the case_studies index.”, which was defined in the index view.

Application View and Models

Most applications will want to render some information when an Http request is made. The views.py contains the views, aka python callback functions, that handle HTTP requests.

They must return an HttpResponse or raise an exception like Http404. In most cases, they will also render to a template but they can also return JSON.

In general, views will render data from the database. In models.py, you define the structure of your data. Models can contain any number of fields. Django has a wide variety of predefined types like boolean, character, date, image, file, etc...

The Admin Interface

The Admin interface is a very useful feature. It's also very simple to use. To register and modify the admin interface for a given field, write you specifications in admin.py. You can configure the order of your fields, group or even omit them.

By default, Django uses meta information from the model to generate an admin interface. For example, if your model has an image field with a technically descriptive name, you can also give it a user friendly name via the verbose_name attribute as well as give the user image dimension requirements using the help_text attribute. Both verbose_name and help_text are used in the admin interface.

class Post(models.Model):
    class Meta:
        verbose_name=_('Blog Post')
        verbose_name_plural=_('Blog Posts')

    tile_image = FilerImageField(
        verbose_name=_('Listing Image'),
        blank=True,null=True,
        help_text='Image size: 400x400',
    )

Configuring the Admin

With Django, you can modify the appearance of the admim like the order and grouping of fields. In admin.py, create a new admin that inherits from ModelAdmin. Here you can alter the admin to your exact specifications. Remember to register it with the site.

class WordAdmin(admin.ModelAdmin):
       #list of groups (aka fieldsets) and the fields in each group.
    _fieldsets = [
        (None, {
            'fields': ['title', ('category','tags'), 'abstract', 'list_image']
        }),
        ('Header Images', {
            'fields': ['header_desktop_image', 'header_tablet_image'],
        }),
        ('Publishing', {
            'fields': (('status'), ('date_published')),
        }),
    ]
    readonly_fields = ['page_link']

#registering the WordAdmin as the admin for the Word Model
admin.site.register(Word, WordAdmin)
